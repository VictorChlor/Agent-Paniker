# Agent-Paniker

## Предсказание панических продаж инвесторов в моменты рыночных кризисов

---

В данном репозитории представлена модель машинного обучения, разработанная для предсказания вероятности панического закрытия позиций (фиксации убытков) частными инвесторами в моменты резкого падения фондового рынка. Проект симулирует реальный сценарий работы аналитического департамента в сфере финтеха и демонстрирует подход к трансформации ML-метрик в превентивные продуктовые решения.

---

## Архитектура данных и спецификация входных признаков

Задача решена в рамках бинарной классификации (где 1 — совершение пользователем панической продажи в течение суток, 0 — отсутствие панических действий) с использованием библиотеки **CatBoostClassifier**.

Модель принимает на вход комбинированный вектор признаков, включающий статические демографические данные, параметры портфеля и сагрегированные поведенческие логи. Для исключения утечки данных все динамические показатели рассчитываются строго за исторический период, предшествующий дню прогноза.

### Перечень признаков, подаваемых на вход модели:

| Имя признака | Тип данных | Описание признака |
| :--- | :--- | :--- |
| **age** | `int` | Возраст пользователя в годах. Используется для учета корреляции между возрастной группой и импульсивностью совершения сделок. |
| **risk_profile** | `string` | Категориальный признак, определяющий инвестиционный психотип клиента (`conservative`, `moderate`, `aggressive`). |
| **experience_days** | `int` | Количество дней с момента открытия брокерского счета. Позволяет модели оценивать уровень рыночного опыта пользователя. |
| **portfolio_value_rub** | `float` | Текущая совокупная стоимость активов на счете клиента в рублях. |
| **stocks_ratio** | `float` | Доля акций в портфеле (в диапазоне от 0.05 до 0.95), определяющая степень чувствительности портфеля к просадкам индекса. |
| **mean_daily_opens** | `float` | Среднесуточное количество сессий в мобильном приложении за предшествующий период (базовая активность). |
| **max_daily_opens** | `int` | Максимальное количество открытий приложения за одни сутки в истории логов клиента (индикатор потенциальной тревожности). |
| **total_screen_views** | `int` | Суммарное количество просмотров экранов приложения за все время существования аккаунта. |
| **total_loss_before_crisis** | `float` | Накопленный финансовый результат пользователя (в рублях) до момента наступления расчетного дня. |

## Методология обучения и валидации 

1. **Обучающая выборка** сформирована на основе признаков, собранных за дни 1–17, и целевой переменной за 18-й день (первая волна рыночного спада).
2. **Валидационная выборка** сформирована на основе признаков за дни 1–18 и целевой переменной за 19-й день (продолжение падения рынка).

Такой подход моделирует реальное применение системы в продакшне, когда алгоритм обучается на ретроспективных кризисных периодах и применяется для прогнозирования поведения клиентов на будущих торговых сессиях.

> **Метрика качества модели на независимом временном отрезке:** ROC-AUC = 0.85

### Продуктовая интеграция и бизнес-сценарии (Концепт)

Поскольку модель обучалась на сгенерированных данных, моделирующих поведение инвесторов, был разработан концепт интеграции этой системы в реальный продакшн мобильного банка. 

Вместо ресурсоемкого инференса на стороне клиента, скоринг рассчитывается на бэкенде в режиме батч-обработки. Полученная вероятность паники (от 0 до 1) передается в приложение для сегментации базы и динамической адаптации UI/UX:

1. **Превентивное информирование:** Пользователям из группы высокого риска в день падения рынка при входе демонстрируется аналитический сторис-блок, разъясняющий цикличность процессов для снижения эмоционального давления.
2. **Коррекция интерфейса сделок:** При попытке продажи резко упавшего актива пользователю с высоким скором паники выводится расчет упущенной выгоды (потеря дивидендов) и зафиксированного убытка для предотвращения необдуманных действий.
3. **Смена фокуса внимания:** При фиксации аномальной частоты сессий (на основе разработанного в проекте признака `max_daily_opens`) акцент на главных экранах смещается с ежесекундного обновления графиков на экспертную текстовую аналитику.
